# Flujo de Desarrollo

**Regla de Oro: Solo cambia lo que se pidió.**

- Busca código existente ANTES de crear algo nuevo.
- Desarrollo incremental: valida que compila en cada paso.
- Si un cambio afecta otros módulos → **notifica al usuario primero y espera aprobación**.
- **PROHIBIDO** modificar archivos o ejecutar comandos destructivos sin autorización explícita en el chat.
- Usa las mismas convenciones (nombres, estilos, patrones) ya presentes en el proyecto.
